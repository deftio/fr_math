API Reference

Every public symbol, grouped by topic. Each entry lists the radix convention, the precision, and the error / saturation behaviour. All types are from FR_defs.h: s8 s16 s32 for signed and u8 u16 u32 for unsigned integers (these are aliases for the <stdint.h> types).

Radix convention. Unless noted otherwise, every function that takes a radix argument interprets all of its fixed-point inputs and outputs at that radix. The library never stores the radix next to the value — it’s your job to track.

Trigonometry — integer degrees (legacy)

The classic FR_Sin / FR_Cos API takes angles as integer degrees in [0, 360). Results are s32 values at the requested radix. All four integer-deg functions use a 90-entry lookup table with quadrant folding.

FunctionSignatureAccuracy
FR_Sins32 FR_Sin(s16 deg, s32 r)±1 LSB at radix 14.
FR_Coss32 FR_Cos(s16 deg, s32 r)±1 LSB at radix 14.
FR_Tans32 FR_Tan(s16 deg, s32 r)Saturates near 90° / 270°.
FR_SinI / FR_CosI / FR_TanIReturn plain s16 integer percentage (0–100).For quick-and-dirty use.

Trigonometry — radian / BAM (v2)

The newer fr_sin / fr_cos / fr_tan family takes angles in radians at a caller-chosen radix, or in BAM (u16, full circle = 65536). These are the recommended APIs for new code because they’re resolution-independent and integrate cleanly with phase accumulators.

FunctionSignatureNotes
fr_sins32 fr_sin(s32 theta, s32 r)Sine of an angle in radians at radix r.
fr_coss32 fr_cos(s32 theta, s32 r)Cosine.
fr_tans32 fr_tan(s32 theta, s32 r)Saturates near π/2 + kπ.
fr_sin_bams32 fr_sin_bam(u16 bam, s32 r)Direct BAM input. No reduction needed.
fr_cos_bams32 fr_cos_bam(u16 bam, s32 r)Direct BAM input.
fr_sin_deg / fr_cos_degs32 fr_sin_deg(s32 deg_fp, s32 r)Degrees as fixed-point (supports fractional degrees).

Accuracy: all radian/BAM functions are bounded by ±2 LSB at radix 14 across the full domain. See dev/fr_math_precision.md for the full table.

Logarithm and exponential

All log / exp functions use the same identity-driven implementation: normalise the input into [1, 2) (or [0, 1) for exp), look up in a small polynomial, and fix up the exponent. Worst-case error is ± 2 LSB at the output radix.

FunctionSignatureDomain / error
FR_log2s32 FR_log2(s32 x, s32 r)x > 0. Returns FR_DOMAIN_ERROR for x ≤ 0.
FR_lns32 FR_ln(s32 x, s32 r)Natural log.
FR_log10s32 FR_log10(s32 x, s32 r)Common log.
FR_pow2s32 FR_pow2(s32 x, s32 r)Saturates for x ≥ 30 at radix 16.
FR_exps32 FR_exp(s32 x, s32 r)Saturates similarly.
FR_pow10s32 FR_pow10(s32 x, s32 r)Saturates around x ≥ 9 at radix 16.

Wave generators

All wave generators take a phase as u16 BAM and return a bipolar amplitude in [−FR_WAVE_PEAK, +FR_WAVE_PEAK]. They are pure functions: no internal state. Drive them from a phase accumulator in your own code.

FunctionSignatureShape
fr_wave_sqrs16 fr_wave_sqr(u16 phase)50% square.
fr_wave_pwms16 fr_wave_pwm(u16 phase, u16 duty)Variable-duty square. duty is a u16 threshold (32768 = 50%).
fr_wave_tris16 fr_wave_tri(u16 phase)Triangle.
fr_wave_saws16 fr_wave_saw(u16 phase)Ramp saw (ascending).
fr_wave_tri_morphs16 fr_wave_tri_morph(u16 phase, u16 sym)Triangle with adjustable peak position. sym is the BAM position of the peak.
fr_wave_noises16 fr_wave_noise(u32 *state)Xorshift-based white noise. Caller owns the 32-bit state.

ADSR envelope

The envelope generator is a 4-state machine: Attack → Decay → Sustain → Release. It is sample-rate agnostic — times are expressed in samples, not seconds.

FunctionSignatureNotes
fr_adsr_initvoid fr_adsr_init(fr_adsr_t *env, u32 a, u32 d, s16 s, u32 r)Set attack/decay/release times (in samples) and sustain level (bipolar amplitude).
fr_adsr_triggervoid fr_adsr_trigger(fr_adsr_t *env)Note-on: moves state to Attack.
fr_adsr_releasevoid fr_adsr_release(fr_adsr_t *env)Note-off: moves state to Release from wherever it is.
fr_adsr_steps16 fr_adsr_step(fr_adsr_t *env)Advance one sample. Returns the current amplitude.

2D transforms (FR_math_2D.h)

FR_Matrix2D_CP is a C++ class wrapping a 3×3 s32 matrix at a configurable radix. It covers the standard 2D graphics operations. The class uses no dynamic allocation — all state is inline.

Formatted output

FunctionSignature
FR_printNumDPretty-print an s32 as signed decimal.
FR_printNumFPretty-print an s32 as fixed-point at a given radix (e.g. “3.14159”).
FR_printNumHPretty-print an s32 as hex.

All three take a user-supplied putc callback so you can direct output at a UART, an in-memory buffer, or stdout without pulling in printf.
